# GY24 QRエントリー＆抽選アプリ — 公開手順

会場のスクリーンにQRを表示 → 参加者が各自スマホで読み取り → 名前・好物を入力して応募
→ 主催者がルーレットで1名を抽選、という流れのWebアプリです。

- **参加者用ページ**: `/`（QRのリンク先。名前＋好物のフォーム）
- **主催者用ページ**: `/host`（QR表示・登録状況のリアルタイム表示・ルーレット抽選）
- QRは開いた**公開URLを自動で読み取って生成**されるので、どこに公開してもそのまま動きます。

---

## 方法A：自分のPCから即公開（推奨・無料・登録不要）

会場にPCがあるなら、これが一番簡単です。Cloudflareのトンネルで一時的なhttps URLを発行します。

1. このフォルダでサーバーを起動
   ```
   node server.js
   ```
   → `http://localhost:3200` で起動します。

2. 別のターミナルで公開トンネルを起動（インストール不要）
   ```
   npx cloudflared tunnel --url http://localhost:3200
   ```
   → `https://xxxx-xxxx.trycloudflare.com` のようなURLが表示されます。

3. **会場の画面では、その公開URLに `/host` を付けて開く**
   例：`https://xxxx-xxxx.trycloudflare.com/host`
   → 表示されたQRは自動的に `https://xxxx-xxxx.trycloudflare.com/` を指すので、参加者はスマホで読み取るだけで応募できます。

> ⚠️ 必ず**公開URLの方で** `/host` を開いてください（localhost で開くとQRが localhost を指してしまい、スマホから繋がりません）。

---

## 方法B：常時公開のクラウドにデプロイ（Render など）

PCを起動し続けなくてよい方法です。例として Render（無料枠あり）：

1. このフォルダを GitHub リポジトリにアップロード
2. [render.com](https://render.com) → New → **Web Service** → リポジトリを選択
3. 設定：
   - **Build Command**: （空でOK。依存パッケージなし）
   - **Start Command**: `npm start`
   - **Instance Type**: Free
4. デプロイ完了後の `https://xxxx.onrender.com/host` を会場画面で開く

> 補足：無料枠はアクセスが無いと一時停止します。イベント中は `/host` を開いたままにしておけば稼働し続けます。
> データ（応募一覧）はサーバー再起動で消えることがあります。イベント単位の利用なら問題ありません。

他に Railway / Glitch / Fly.io などでも `npm start` で同様に動きます。

## 主催者向けの操作
- **抽選**：`/host` の「🎰 抽選スタート！」を押すとルーレットが回り1名が当選します。
  「当選済みの人を除外する」で連続抽選も可能。
- **CSV書き出し**：応募者一覧をCSVで保存できます。
- **リセット**：全エントリーを削除します（本番開始前にテストデータを消すなど）。

### 抽選・リセットを保護したい場合（任意）
参加者に勝手に抽選されないよう、起動時に鍵を設定できます。
```
ADMIN_KEY=好きな文字列 node server.js     # 方法A
```
（Renderなら環境変数 `ADMIN_KEY` を設定）
設定した場合、主催者は `/host?key=好きな文字列` で開いてください。

## ファイル構成
- `server.js` … サーバー本体（Node標準モジュールのみ・依存パッケージゼロ）
- `public/entry.html` … 参加者用フォーム
- `public/host.html` … 主催者用（QR・抽選）
- `public/qrcode.js` … QR生成ライブラリ（同梱）
- `data/entries.json` … 応募データ（自動生成）
- `package.json` … デプロイ用
